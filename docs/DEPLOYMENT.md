# Solar Path — Deployment (Render + Neon Postgres)

Production URL: **https://solar-path.onrender.com**

---

## 1. Create Neon Postgres (free)

1. Sign up at [neon.tech](https://neon.tech).
2. Create a project (e.g. `solar-path`).
3. Create database `solarpath` (default is fine).
4. Copy the **connection string** (pooled recommended for serverless):
   ```
   postgresql://USER:PASSWORD@HOST/neondb?sslmode=require
   ```
5. If the URL starts with `postgres://`, the app normalizes it to `postgresql://` automatically.

---

## 2. Set `DATABASE_URL` on Render

### Option A — Dashboard (manual)

1. Open [Render Dashboard](https://dashboard.render.com) → service **solar-path**.
2. **Environment** → add or edit:
   - Key: `DATABASE_URL`
   - Value: *(paste Neon connection string)*
3. **Save** → Render redeploys automatically.

### Option B — Script (recommended)

Add to local `.env` (never commit):

```env
RENDER_API_KEY=rnd_...          # Render → Account Settings → API Keys
DATABASE_URL=postgresql://...    # Neon connection string
```

Then run:

```bash
python scripts/setup_neon_render.py
```

This tests the database connection, pushes `DATABASE_URL` to Render, and triggers a redeploy.

---

## 3. Verify

After deploy (~2 min):

```bash
curl https://solar-path.onrender.com/health
```

Expect:

```json
"database": { "ok": true, "url": "HOST/neondb?sslmode=require" }
```

The `url` field must **not** be `"sqlite"` on production.

## 4. Other production secrets

Set in Render dashboard only (`sync: false` in `render.yaml`):

| Variable | Purpose |
|----------|---------|
| `DATABASE_URL` | Neon Postgres connection string |
| `SMTP_PASSWORD` | Gmail app password (not login password) |
| `BETA_ACCESS_PASSWORD` | Optional closed-beta gate |
| `STRIPE_SECRET_KEY` | When enabling live Stripe |

`SECRET_KEY` and `ADMIN_TOKEN` are auto-generated by the Render blueprint.

---

## 5. SMTP (Gmail)

1. Enable 2FA on the Gmail account.
2. Create an [App Password](https://myaccount.google.com/apppasswords).
3. Set in Render environment:
   - `SMTP_HOST=smtp.gmail.com`
   - `SMTP_PORT=587`
   - `SMTP_USER=your@gmail.com`
   - `SMTP_PASSWORD=<16-char app password>`
   - `SMTP_USE_TLS=1`
   - `SMTP_FROM=your@gmail.com`

Check `/health` → `"email": { "ready": true }`.

## 6. Local vs production database

| Environment | `DATABASE_URL` | Storage |
|-------------|----------------|---------|
| Local dev | unset | SQLite `data/solarpath.db` |
| Docker / Codespaces | `postgresql://...` | Postgres container |
| Render production | Neon URL | Postgres (persistent) |

On first start with an empty database, `init_db()` creates tables and seeds suppliers from `data/suppliers.json`.

---

## 7. Troubleshooting

| Symptom | Fix |
|---------|-----|
| `/health` shows `"url": "sqlite"` on Render | `DATABASE_URL` not set — complete step 2 |
| `/health` database `ok: false` | Wrong password, missing `?sslmode=require`, or Neon project paused |
| Data lost after redeploy | Was using SQLite on ephemeral disk — switch to Neon |
| Cold start 30–60s | Render free tier spins down when idle |
| Email `ready: false` | Set all `SMTP_*` vars in dashboard |
